Two-Phase Payouts

prepare_payout runs VOP + balance check, returns a preparation_id. No money moves. confirm_payout fires the transfer after user confirmation. Preparations expire after 5 minutes.

Fuzzy Reconciliation

Exact match first, then case-insensitive partial reference matching with amount tolerance. Amounts over 1000 require exact match. Fuzzy matches over 10k are flagged for manual review.

FX Slippage Protection

Pre-trade quote → execute → compare actual rate vs quoted rate. Warns if slippage exceeds 0.5%. Uses integer-cent math to avoid floating-point drift.

Treasury Report

Parallel fetch of accounts, balances, payouts, transactions. Flags: zero balances, stale pending payouts (>24h), failed payouts, large transactions (>10k).

Dual API Architecture

  • Banking REST · Bearer JWT · accounts, payouts, FX, deposits
  • Payments All POST · API key · checkout, VOP, refunds, banks
  • Payments API is optional — agent degrades gracefully without it

Safety

  • Idempotency keys on all write operations
  • Streaming responses via SSE (no silent waits)
  • VOP failure → skip, don't block
  • Retry tool validates status: failed before re-sending
  • 19 tests covering failure paths

Tool Use Loop

User message → Claude decides which tools to call → tool handler executes against Augustus APIs → results fed back to Claude → Claude calls more tools or responds. The loop repeats until Claude produces a final text response. History capped at 40 messages with sliding window.
